# Visual Reference System

## Overview

The Visual Reference System (mood board) collects and organizes reference material for artists, audio designers, and the entire team to maintain consistent aesthetic direction. It exists because vague instructions like "make it look medieval" produce wildly different interpretations.

## Purpose

1. **Align the team** on what the game should look and sound like
2. **Reference first** -- check existing references before creating new art or audio
3. **Track evolution** -- update as visual direction develops
4. **Specificity** -- cite exactly what to borrow from each reference

## How to Use

### Adding References

1. Save image files in `assets/art/reference/` with clear names (e.g., `ref_firelink-shrine-lighting.jpg`)
2. Add entries to the appropriate reference table below
3. Write 1-2 sentences describing what specifically is being referenced and why
4. Note which documents it influences

### Reference Categories

| Category | What It Captures |
|----------|-----------------|
| **Art Style Direction** | Primary visual style (pixel art, hand-drawn, low-poly, etc.) |
| **Lighting & Atmosphere** | Mood-defining lighting setups and color temperature |
| **UI Interface Style** | Menu, HUD, and interface visual direction |
| **Character Design** | Armor detail level, facial style, silhouette readability |
| **Environment & Architecture** | Building style, stone/wood textures, scale references |
| **Animation Style** | Frame count, attack speed, idle animations, weight |

## Suggested Reference Sources

- **Games** -- direct gameplay references for mechanics and feel
- **Pinterest** -- shared mood boards for the team
- **ArtStation** -- professional game art examples
- **YouTube** -- gameplay footage for animation and feel
- **Museum databases** -- authentic medieval art and architecture
- **Personal photos** -- real medieval buildings, landscapes, castles

## Anti-References

Equally important: defining what the game should NOT look or sound like. This prevents common pitfalls and aligns expectations early.
