# Level Design Principles

## Overview

Level design principles for the Feudal game project, based on the level design document template and informed by feudal game research. This page covers the core approach to designing game areas, layouts, and player journey through the world.

## Level Documentation Template

Each game area/level should be documented with:
- **Type**: Hub / Combat / Puzzle / Narrative / Mixed
- **Size**: Small / Medium / Large
- **Status**: Planned / In Progress / Complete

### Per-Area Documentation
1. **Layout description**: Physical layout, key landmarks, and player flow
2. **Key Points of Interest**: Named landmarks with type and description
3. **Enemy encounters**: Location, enemy type, difficulty, patrol/behavior notes
4. **Puzzles / challenges**: Environmental problems the player must solve
5. **Secrets / optional content**: Hidden areas, collectibles, optional encounters
6. **Visual mood & atmosphere**: Lighting, color palette, weather, mood

## Level Progression

Levels should connect with a clear intended player journey documented as:
- Order/sequence
- Area name
- What unlocks access to each area
- What each area unlocks (gating)

## Feudal-Informed Level Design

Based on research, feudal game levels should consider:
- **[[Manor]] as focal point**: The lord's seat as a central hub (per Manor Lords' success)
- **[[Castle]] networks**: Castles as administrative centers controlling surrounding territory
- **Settlement hierarchy**: Village → Town → Castle → Keep, each offering different gameplay
- **Organic growth**: Levels that evolve over time based on player decisions (as in Manor Lords)
- **Regional identity**: Different areas with distinct visual styles, resources, and threats

## Design Principles from Research

1. **Visual feedback**: Players respond to seeing their domain grow visually, not just numerically
2. **Meaningful exploration**: Each area should have points of interest that reward investigation
3. **Atmospheric consistency**: Visual mood should reinforce the feudal setting (grim castles, busy villages, dark forests)
4. **Progressive complexity**: Start small (manor/village) and expand outward
5. **Emergent discovery**: Hidden areas and secrets that tell stories about the world
