# Game Mechanics Overview

## Overview

Master index of all game mechanics planned or researched for the Feudal game project, combining current design decisions with research-informed recommendations.

## From Design Document (Current Draft)

The design document outlines these mechanical categories, currently in placeholder status:

- **Player Mechanics**: Movement, interaction, combat, inventory
- **World Mechanics**: [[Feudal system]], [[NPC]] behavior, economy, day/night cycle
- **UI Systems**: Main menu, HUD, inventory screen, settings
- **Input**: Keyboard (WASD/E), gamepad, mouse mappings

## From Research (Recommended)

Based on comprehensive analysis of 18 feudal games, the following mechanics are recommended:

### Tier 1 — Core (P0)
- [[Vassal loyalty systems|Vassal loyalty/rebellion]] — Core feudal tension
- [[Land management|Land/demesne management]] — The feudal power principle

### Tier 2 — Important (P1)
- Succession & inheritance — Story arc generation
- Economy — Simplified, meaningful chains

### Tier 3 — Supporting (P2)
- Court and council system — Where feudal drama plays out
- Faction system — Group vassals around shared interests

### Tier 4 — Enhancement (P3)
- Military/defense — Secondary to social dynamics
- Religion/church relations — Depth addition

## Mechanics to Avoid
- Complex production chains — Use short, meaningful chains
- Micromanagement of individual buildings — Organic placement preferred
- Spreadsheet-only army combat — Name commanders at minimum
- Single-axis loyalty systems — Use 2-3 factors minimum
- Endless resource accumulation — Add meaningful sinks
