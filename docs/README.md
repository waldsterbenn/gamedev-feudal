# Feudal - Game Development Workspace

> **Project:** Feudal
> **Founded:** 2026-04-05
> **Status:** Pre-Production

## Workspace Structure

```
gamedev-feudal/
├── docs/
│   ├── design/
│   │   ├── game-design.md       # Core game design document
│   │   ├── world-lore.md        # World building and lore bible
│   │   └── level-design.md      # Level layouts and area documentation
│   ├── tech/
│   │   └── technical-spec.md    # Engine, architecture, and tech specs
│   ├── art/
│   │   └── art-style-guide.md   # Visual style and art guidelines
│   ├── audio/
│   │   └── audio-design.md      # Music, SFX, and audio specs
│   ├── qa/
│   │   └── testing-plan.md      # QA strategy and bug tracking
│   └── project/
│       ├── project-plan.md      # Milestones, team, and sprint planning
│       └── changelog.md         # Project change history
├── src/          # Source code (create when ready)
├── assets/       # Game assets (create when ready)
├── config/       # Config files (create when ready)
└── build/        # Build outputs (create when ready)
```

## Quick Start

1. **Fill in the game design doc** -- Start with `docs/design/game-design.md`
2. **Define your world** -- Expand `docs/design/world-lore.md`
3. **Pick your tech stack** -- Update `docs/tech/technical-spec.md`
4. **Set your art direction** -- Configure `docs/art/art-style-guide.md`
5. **Plan your scope** -- Set milestones in `docs/project/project-plan.md`

## Conventions

- All documentation uses Markdown (`.md`)
- One document per concept (keep files focused)
- Status tags: `Draft` / `In Review` / `Approved` / `Archived`
- Tables use pipe syntax with header separators
